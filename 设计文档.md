# AI 代码仓扫描结果管理平台 - 产品设计文档

本文档面向 `aiRepoScan-hw` 前端方案，按“业务需求与功能设计 -> 系统设计与需求分析 -> 数据表与接口设计”的顺序整理，并补充 DFX 评分与场景功能测试，便于评审与落地实施。

---

## 1. 业务需求分析与功能设计

### 1.1 业务背景与问题

团队在代码仓执行静态分析或 AI 扫描后，会生成大量告警数据（文件、行号、规则、说明、代码片段）。仅保留原始 JSON 难以满足以下管理诉求：

- 以任务维度管理扫描结果和处理进度；
- 多人协作标注告警结论并沉淀原因；
- 按任务统计完成度、规则命中和治理趋势。

### 1.2 产品定位与价值

| 维度 | 说明 |
|------|------|
| 定位 | 代码仓扫描结果管理与标注工作台，与扫描引擎解耦 |
| 核心价值 | 将分散告警转为可检索、可标注、可统计的数据资产 |
| 业务边界 | 不负责执行扫描；依赖后端提供任务、结果文件和统计接口 |

### 1.3 用户角色与业务场景

| 角色 | 关键诉求 |
|------|----------|
| 任务负责人 | 创建任务、关联结果、查看任务状态和规模 |
| 标注人员 | 按规则和关键字筛选告警，提交标注与原因 |
| 管理者 | 查看完成率、状态分布、规则 Top，驱动质量治理 |

核心业务流程：创建任务 -> 上传/同步扫描结果 -> 任务完成 -> 进入详情标注 -> 统计看板复盘。

### 1.4 功能设计

#### 1.4.1 总览页（`OverviewView.vue`）

- 目标：一眼查看全局任务与缺陷态势。
- 规划功能：任务状态分布、缺陷趋势、近期任务列表。
- 当前状态：已完成页面布局占位，待接入真实聚合接口。

#### 1.4.2 任务管理（`TaskListView.vue`）

| 功能点 | 设计说明 |
|--------|----------|
| 任务视图切换 | 支持“全部任务 / 我的任务” |
| 筛选 | 任务名称模糊搜索、任务状态筛选、重置 |
| 列表展示 | 任务名、状态、仓库、分支、创建人、创建时间、助手版本、扫描路径 |
| 分页 | 支持页码/页大小切换并显示总数 |
| 创建任务 | 使用 `CreateTaskDialog`，校验必填项与 URL 格式 |
| 删除任务 | 仅创建人可删，进行中任务禁止删除 |
| 查看详情 | 跳转 `/task/:id` |

#### 1.4.3 任务详情-任务信息（`TaskDetailView.vue`）

| 功能点 | 设计说明 |
|--------|----------|
| 查看与编辑 | 支持查看模式和编辑模式切换 |
| 元数据维护 | 可修改任务名、仓库、分支、路径、状态、产品等字段 |
| 文件上传 | 支持上传扫描结果 JSON，触发详情与结果刷新 |

#### 1.4.4 任务详情-标注视图（`TaskDetailView.vue`）

| 功能点 | 设计说明 |
|--------|----------|
| 告警列表 | 展示文件、规则、行号、描述、代码片段 |
| 规则树 | 按规则聚合并支持关键字检索 |
| 标注操作 | 需要修改 / 无需修改 / 问题误报，支持填写原因与取消 |
| 筛选分页 | 按标注状态、关键词过滤，分页请求保证性能 |

#### 1.4.5 任务详情-统计可视化

| 统计项 | 设计说明 |
|--------|----------|
| 总缺陷数 | 当前任务告警总数 |
| 标注完成度 | 已标注/未标注数量与完成率，环形图展示 |
| 标注状态分布 | 需要修改/无需修改/问题误报/未标注，饼图展示 |
| 规则分布 | 按规则聚合柱状图（Top N） |

### 1.5 体验目标（验收视角）

| 场景 | 期望效果 |
|------|----------|
| 新用户进入 | 快速理解导航并完成任务创建或定位 |
| 单条告警处理 | 低学习成本完成标注和原因填写 |
| 负责人查看进度 | 快速看到完成率与规则分布 |
| 大规模告警 | 通过分页、筛选、规则树高效定位 |
| 权限场景 | 删除权限清晰，避免误操作 |

## 2. 系统设计与需求分析

### 2.1 需求分析（系统侧）

| 需求类型 | 需求内容 |
|----------|----------|
| 功能需求 | 任务创建、任务查询、任务编辑、结果上传、告警标注、统计展示 |
| 性能需求 | 告警大数据量场景下保持可用，采用服务端分页 |
| 可用性需求 | 关键路径（创建、标注、保存）操作反馈明确 |
| 安全与权限 | 删除受创建人约束，接口依赖登录态和后端鉴权 |
| 可扩展性 | 支持后续“专家评审”流程扩展而不破坏主流程 |

### 2.2 逻辑架构（前端视角）

```text
[浏览器]
  ├─ 布局层：顶栏 + 侧栏 + 内容区
  ├─ 页面层：总览 / 任务列表 / 任务详情
  ├─ 状态层：Pinia（用户信息）
  └─ 数据层：API 封装（Mock 或 HTTP）
                ↓
[后端服务 + 存储]
  REST API + 对象存储(S3) + 业务数据库
```

### 2.3 技术选型

| 类别 | 选型 | 用途 |
|------|------|------|
| 框架 | Vue 3 | 组件化与响应式 |
| 路由 | Vue Router | 页面路由管理 |
| 状态 | Pinia | 全局用户状态 |
| UI 组件 | Element Plus | 表单、表格、分页、弹窗、标签 |
| 图表 | ECharts | 环形图、饼图、柱状图 |
| 高亮 | highlight.js | 代码片段高亮展示 |
| 构建 | Vite | 开发与构建 |

### 2.4 信息架构与路由

| 路由 | 页面 | 说明 |
|------|------|------|
| `/` | 重定向 | 默认进入总览 |
| `/overview` | 总览页 | 平台级汇总入口 |
| `/tasks` | 任务管理页 | 创建、筛选、查看任务 |
| `/task/:id` | 任务详情页 | 信息、标注、统计一体化 |

### 2.5 权限与非功能设计

| 项目 | 设计说明 |
|------|----------|
| 权限控制 | 删除任务仅创建人可见，进行中任务禁删 |
| 数据一致性 | 标注保存后更新列表与统计，口径以后端统计为准 |
| Mock 联调 | `src/api/task.ts` 支持本地模拟和 localStorage 持久化 |
| 环境配置 | 通过 `VITE_API_REPO_SCAN` 等变量切换环境 |
| 扩展规划 | 预留专家评审能力：评审状态、评审人、评审时间、意见 |

### 2.6 通用响应规范

接口采用统一信封结构：`{ data, meta }`。  
成功：`meta.isSuccess = true`；失败：通过 `meta.message`、`meta.number` 返回错误信息。

## 3. 数据表与接口设计

### 3.1 核心数据实体（逻辑表）

#### 3.1.1 任务表 `task_info`

| 字段 | 类型 | 说明 |
|------|------|------|
| `task_id` | string | 主键，任务 ID |
| `task_name` | string | 任务名称 |
| `task_status` | string | 未开始/进行中/已完成/失败 |
| `repo_url` | string | 代码仓地址 |
| `branch` | string | 扫描分支 |
| `path_list` | string | 扫描路径（逗号分隔） |
| `assistant_versions` | string | 助手版本串 |
| `product_name` | string | 产品名 |
| `code_language` | string | 代码语言 |
| `line_num` | number | 代码量 |
| `creator` | string | 创建人工号 |
| `name_cn` | string | 创建人中文名 |
| `dept_name` | string/null | 部门 |
| `pdu_name` | string/null | PDU |
| `s3_path` | string | 结果文件路径 |
| `warn_count` | number | 告警总数 |
| `create_time` | datetime | 创建时间 |

#### 3.1.2 告警明细表 `scan_result`

| 字段 | 类型 | 说明 |
|------|------|------|
| `warn_uuid` | string | 主键，告警唯一 ID |
| `task_id` | string | 关联任务 ID |
| `file_name` | string | 文件路径 |
| `function_name` | string | 函数名 |
| `rule_name` | string | 规则名 |
| `warn` | string | 告警说明 |
| `warn_line` | number | 告警行 |
| `start_line`/`end_line` | number | 代码片段行范围 |
| `warn_code_block` | string | 告警代码块 |
| `code_snippet` | string | 代码切片 |
| `context` | string | 上下文 |
| `confidence` | number | 置信度 |
| `issue_result` | number/null | 当前标注结论 |
| `reason` | string | 标注原因 |

#### 3.1.3 标注表 `annotation_info`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | number | 主键 |
| `task_id` | string | 任务 ID |
| `warn_uuid` | string | 告警 ID |
| `user_id` | string | 标注人工号 |
| `user_name` | string/null | 标注人姓名 |
| `issue_result` | number | 0需要修改/1无需修改/2问题误报 |
| `reason` | string/null | 标注原因 |
| `annotation_status` | number | 标注状态 |
| `create_time` | datetime | 创建时间 |
| `update_time` | datetime | 更新时间 |

### 3.2 接口设计

| 能力 | 前端函数（`src/api/task.ts`） | 方法与路径 |
|------|------------------------------|------------|
| 查询任务列表 | `queryTaskList` | `GET /api/tasks` |
| 查询任务信息 | `getTaskInfo` | `GET /api/tasks/{taskId}/info` |
| 查询扫描结果 | `getTaskScanResults` | `GET /api/tasks/{taskId}/scan-results` |
| 创建任务 | `createTaskApi` | `POST /api/tasks` |
| 更新任务 | `updateTaskInfo` | `PUT /api/tasks/{taskId}` |
| 删除任务 | `deleteTaskById` | `DELETE /api/tasks/{taskId}` |
| 上传结果文件 | `uploadScanResultFile` | `POST /api/tasks/{taskId}/uploadDataSet` |
| 查询标注统计 | `getAnnotationStatistics` | `GET /api/tasks/{taskId}/annotation-statistics` |
| 保存标注 | `saveAnnotationApi` | `POST /api/tasks/{taskId}/annotations`（以实际部署为准） |

### 3.3 关键接口入参与出参要点

- 创建任务：必填任务名、产品名、仓库 URL、分支；可选路径、版本、语言、组织信息。
- 扫描结果查询：支持分页与筛选条件（标注状态、关键词、规则）。
- 保存标注：支持 `issueResult` 为 `null` 表示取消标注。
- 统计查询：返回总数、完成度、状态分布、规则分布，用于图表渲染。

### 3.4 标注结果枚举

| `issueResult` / `issue_result` | 含义 |
|--------------------------------|------|
| `null` | 未标注 |
| `0` | 需要修改 |
| `1` | 无需修改 |
| `2` | 问题误报 |

## 4. DFX 评分与场景功能测试

### 4.1 DFX 评分

> 评分口径：满分 100 分，按 D(可开发性) / F(功能可用性) / X(可扩展与可运维性) 三维评估。

| 维度 | 权重 | 当前得分 | 说明 |
|------|------|----------|------|
| D - 可开发性 | 30 | 26 | 模块边界清晰，类型定义较完整，少量接口契约仍需与后端收敛 |
| F - 功能可用性 | 40 | 34 | 核心链路完整（任务->标注->统计），总览页仍为占位态 |
| X - 可扩展与可运维性 | 30 | 25 | 已预留专家评审扩展点，监控与异常治理文档可继续补齐 |
| 总分 | 100 | 85 | 具备上线基础，建议完成总览聚合接口后进入正式验收 |

### 4.2 场景功能测试

| 测试场景 | 前置条件 | 测试步骤 | 预期结果 |
|----------|----------|----------|----------|
| 创建任务成功 | 用户已登录 | 打开任务列表 -> 点击创建任务 -> 填写必填并提交 | 创建成功，列表出现新任务 |
| 创建任务校验 | 用户已登录 | 创建任务时输入非法仓库 URL 或缺少必填项 | 阻止提交并提示校验信息 |
| 删除任务权限 | 存在他人创建任务 | 当前用户查看任务列表 | 非创建人不显示删除按钮 |
| 删除进行中任务 | 存在进行中且本人创建任务 | 点击删除 | 删除按钮禁用或提示不可删除 |
| 上传扫描结果 | 任务详情可访问 | 在任务信息页上传 JSON 文件 | 上传成功并刷新告警与统计 |
| 标注告警 | 任务状态已完成且有告警 | 进入标注页 -> 选择结论 -> 填写原因 -> 提交 | 标注成功，卡片状态更新 |
| 取消标注 | 已标注告警存在 | 点击取消标注并确认 | 结论恢复未标注，统计同步变化 |
| 筛选与分页 | 告警数量大于一页 | 设置标注状态与关键词，切换页码 | 列表按条件过滤且分页正确 |
| 统计正确性 | 存在多种标注结论 | 打开统计区检查总数、完成率、分布图 | 图表与明细数据口径一致 |
| 异常提示 | 模拟接口失败 | 执行任务查询/标注保存 | 页面展示失败提示，不出现卡死 |
