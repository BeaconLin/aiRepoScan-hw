# 一、数据定义

## 1、任务列表项（ScanTask）

任务管理页面任务列表中每个任务的类型定义。

| 字段 | 类型 | 说明 |
|------|------|------|
| taskId | string | 任务 ID，以 T 开头的 UUID |
| taskName | string | 任务名称 |
| taskStatus | string | 任务扫描状态：未开始、进行中、已完成、失败 |
| assistantVersions | string[] | 扫描助手名称，可能多个 |
| branch | string | 扫描分支 |
| codeLanguage | string | 主要代码类型，如 C++、Java |
| createTime | string | 任务创建时间，格式 yyyy-mm-dd hh:mm:ss |
| creator | string | 任务创建人短工号，如 a00559876 |
| lineNum | number | 扫描代码量（k，千行） |
| paginationInfo | PaginationInfo | 分页信息，任务列表页时存在 |
| pathList | string | 扫描路径，多个路径用英文逗号分隔，如 `src,main` |
| productName | string | 产品名称，如 UDM |
| repoUrl | string | 扫描代码仓链接 |
| s3Path | string | 扫描结果保存的 S3 路径 |
| warnCount | number | 告警总数，如 1577 |
| scanResults | ScanResultItem[] | 扫描缺陷列表，任务列表页可为空数组 |

## 2、分页信息（PaginationInfo）

| 字段 | 类型 | 说明 |
|------|------|------|
| totalPages | number | 总页数 |
| pageSize | number | 每页条数 |
| hasPrevious | boolean | 是否有上一页 |
| hasNext | boolean | 是否有下一页 |
| currentPage | number | 当前页码 |
| totalCount | number | 总记录数 |

## 3、扫描结果项 / 告警项（ScanResultItem）

单条扫描缺陷/告警的数据结构。

| 字段 | 类型 | 说明 |
|------|------|------|
| warn_uuid | string | 缺陷标识 ID（UUID） |
| warn_line | number | 告警在本文件的代码行 |
| warn | string | 告警描述信息 |
| warn_code_block | string | 告警代码块 |
| check_function_id | string \| null | 所属函数 ID，可为 null |
| code_snippet | string | 切片代码块 |
| confidence | number | 判断概率 |
| context | string | 告警的上下文代码 |
| start_line | number | 起始代码行 |
| end_line | number | 结束代码行 |
| file_name | string | 缺陷所在文件名 |
| function_name | string | 函数名称，如 ASN1_AddTagAndLen |
| func_uuid | string | 函数 UUID |
| self_increment_id | number | 自增 ID |
| index | number \| null | 第几条缺陷，可为 null |
| reason | string | 用户标记的原因说明 |
| rule_name | string | 规则名称，如 内存安全 |
| annotation | Annotation \| null | 标注信息，未标注时为 null |

## 4、标注信息（Annotation）

单条告警的标注记录。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | number | 标注记录 ID |
| warnUuid | string | 告警 UUID，与 warn_uuid 对应 |
| userId | string | 标注用户短工号，如 t00598420 |
| issueResult | number | 标注结果：0-需要修改，1-无需修改，2-问题误报 |
| reason | string \| null | 标注原因说明 |
| annotationStatus | number | 标注状态：1-已标注 |
| createTime | string | 标注创建时间，yyyy-mm-dd hh:mm:ss |
| updateTime | string | 标注更新时间，yyyy-mm-dd hh:mm:ss |
| userName | string \| null | 用户姓名 |
| userDepartment | string \| null | 用户部门 |
| taskId | string \| null | 任务 ID |

## 5、用户信息（UserInfo）

userProfile 中的用户信息类型。

| 字段 | 类型 | 说明 |
|------|------|------|
| w3Id | string | 用户短工号，姓名首字母+8 位数字，前两位 00，如 z00660606 |
| name | string | 用户长工号，姓名拼音+纯数字工号，如 zhangsan 00660606 |
| nameCn | string | 中文姓名，如 张三 |

### 1.6 创建代码仓扫描任务

**接口描述**：创建代码仓扫描任务，任务创建成功后返回任务 ID。

**请求方式**：`POST /api/tasks`

**请求体（JSON）**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| taskName | string | 是 | 任务名称 |
| productName | string | 是 | 产品名称 |
| repoUrl | string | 否 | 代码仓库地址 |
| branch | string | 否 | 分支名称 |
| pathList | string | 否 | 扫描路径列表，多个路径用英文逗号分隔，如 `path1,path2` |
| creator | string | 否 | 任务创建人 |
| assistantVersions | string | 否 | 扫描助手版本，多个可用逗号分隔，如 `v1,v2` |
| codeLanguage | string | 否 | 代码语言，如 Java |
| deptName | string | 否 | 部门名称 |
| pduName | string | 否 | PDU 名称 |
| lineNum | number | 否 | 扫描代码量（k，千行） |

**请求示例**：

```json
{
  "taskName": "新建任务",
  "productName": "CDN",
  "repoUrl": "https://example.com/repo.git",
  "branch": "master",
  "pathList": "path1,path2",
  "creator": "creator1",
  "assistantVersions": "v1,v2",
  "codeLanguage": "Java",
  "deptName": "部门名称",
  "pduName": "PDU名称",
  "lineNum": 10
}
```

**响应示例**：

成功（HTTP 200）：

```json
{
  "meta": {
    "success": true,
    "message": "success",
    "number": 200
  },
  "data": {
    "taskId": "T00b6b10f-e768-4c72-bd6f-9a087f78c847",
    "taskName": "新建任务",
    "productName": "CDN",
    "taskStatus": "未开始",
    "createTime": "2026-02-27 10:00:00"
  }
}
```

失败 - 任务名称为空（HTTP 400）：

```json
{
  "meta": {
    "success": false,
    "message": "任务名称不能为空",
    "number": 400
  },
  "data": null
}
```

失败 - 产品名称为空（HTTP 400）：

```json
{
  "meta": {
    "success": false,
    "message": "产品名称不能为空",
    "number": 400
  },
  "data": null
}
```

失败 - 任务创建失败（HTTP 500）：

```json
{
  "meta": {
    "success": false,
    "message": "任务创建失败",
    "number": 500
  },
  "data": null
}
```

### 1.7 根据任务 ID 删除任务并清除缓存

**接口描述**：根据任务 ID 删除任务信息，同时清除该任务的所有缓存数据。

**请求方式**：`DELETE /api/tasks/{taskId}`

**路径参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| taskId | string | 是 | 任务 ID |

**响应示例**：

成功（HTTP 200）：

```json
{
  "meta": {
    "success": true,
    "message": "success",
    "number": 200
  },
  "data": "任务删除成功"
}
```

失败 - 任务 ID 为空（HTTP 400）：

```json
{
  "meta": {
    "success": false,
    "message": "任务ID不能为空",
    "number": 400
  },
  "data": null
}
```

失败 - 任务不存在或删除失败（HTTP 500）：

```json
{
  "meta": {
    "success": false,
    "message": "任务删除失败或任务不存在",
    "number": 500
  },
  "data": null
}
```

失败 - 其他错误（HTTP 500）：

```json
{
  "meta": {
    "success": false,
    "message": "删除任务失败: error message",
    "number": 500
  },
  "data": null
}
```

**注意事项**：

- 删除任务时会自动清除该任务的所有缓存数据。
- 删除操作不可恢复，请谨慎使用。
- 只能删除已存在的任务；若任务不存在，将返回失败。
